一直想搭建自己的个人网站，前些天了解了 Hexo 建站大法，捯饬了一番竟然成功了，记录过程以免大家重复趟坑。我所用的设备是 Mac ，网站部署至 github 和 阿里云虚拟主机。

Hexo 建站流程

使用 Hexo 没有其它原因，只是相对方便，并且搭出来的网站效果不错。官方对 Hexo 的解释是：

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

刚接触的时候看这段话仍然对 Hexo 一知半解，折腾完之后才摸出个大概门道。Hexo 建站过程可以用下图简单示例：

用文字描述就是，利用 Markdown 语法编辑文章，编辑完成后使用 Hexo 提供的主题生成静态页面，然后将文件上传至github等服务器，最后将域名指向该服务器地址。下面详细说明建站过程。

二、配置主题

更换主题

1 挑选主题
在hexo官网上挑选喜欢的主题，点击主题名称打开github地址即可得到主题文件。建议选择star数较多的主题，这样问题会比较少。

2 保存主题至本地
假如你选择了next主题，通过任何方式将主题下载至themes文件夹即可。主题文件夹一般以一个单词命名，如 next, landscape。

3 配置_config.yml文件
在网站根目录配置文件_config.yml 中将主题配置为你所下载的文件名称，如theme: next。

4 体验主题
更换主题后，同样执行 hexo g 和 hexo s 来体验主题，如果有错误，注意检查以下两项：

  • 下载的主题是不是放在了themes文件夹下面，且主题文件夹的名称是否是minos。
  • 站点配置_config.yml中的主题名称是否正确。
    如果这两项都没问题，也许是主题本身有问题，建议换个主题试试。

自定义主题

如果不满意主题中的某些内容或者样式，我们可以选择自定义主题。在跳进去瞎搞之前，我们需要大概了解下主题是怎么起作用的。这部分建议阅读下 hexo 官网的主题说明。

到这儿你可能会疑惑，post和page之间有什么区别？我从实践得出的结论是，所有文章格式都是post，你所创建的post会出现在archive（归档）列表中，而page是则不会出现在文章列表中，它是像about这样单独的页面。

创建文件还有很多细节可以定义，比如tags, date等，因为我了解不深这里就不细说了，以后遇到再细说。

此外，由于 hexo 文章都是用markdown语法写的，有必要了解一下相关语法。至于编辑器，我用的是 Typora，感兴趣的可以下载试试。

四、部署至服务器

执行了 hexo g 之后就会在 public 文件夹中生成网站的静态网页，要让其他人也能访问我们的网站，我们需要将这些网页部署（上传）至远程服务器。
目前我了解到的常用远程服务器有两种：github 服务器和腾讯云/阿里云服务器。由于我之前强迫症犯了没忍住在阿里云上买了云虚拟主机，所以两者均需部署，下面我分别记录下部署过程。

五、域名配置

如果对域名没要求，直接用 github page 做域名也可以。如果想用自己的域名，那也只需要两步就可搞定：

1.购买域名
具体操作网上一大堆资料，我就不多说了。一点建议是，刚接触博客就别购买太贵的域名了，毕竟多数人折腾博客纯属自娱自乐，又没多大流量。

2.域名解析
购买了自己的域名，需要将该域名解析至我们的服务器。如果我们用的服务器是 github ，在存放网站的 repositories 下的 setting 页面中，找到 github pages ，在 custom domain 中填上自己的域名并保存即可。

如果是在万网上购买的，在阿里云域名管理中，点击解析，按照操作指引将域名解析至存放网站的服务器即可。
