Hexo是什么？

简单的说，Hexo是一个由 Node.js 驱动的一款快速、简单且功能强大的博客框架。它的主要作用是，使用 Markdown（或其他渲染引擎）解析文章，在几秒内利用自带主题生成静态网页。它的作用是让使用者专注写作，而不是花大量时间配主题。

安装Hexo

Step1 安装Node.js

Node.js是什么？具体参考：https://www.zhihu.com/question/33578075

Mac下如何安装node：
$ brew install node

Step2 安装Git

在终端中输入 git —version, 发现已经安装

Step3 安装Hexo

$ npm install -g hexo-cli

什么是npm？
npm是Nodejs下的包管理器，Use npm to install, share, and distribute code; manage dependencies in your projects; and share & receive feedback with others.

配置Hexo

Step1 进入存放网站内容的本地文件夹
$ cd /folder

Step2 初始化该文件夹
$ hexo init
$ npm install

设置git用户名和邮箱

  1. 为什么要设置？
    用户名和邮箱地址是本地git客户端的一个变量，它的作用是告诉git每次commit都是由谁来执行的。github的contributions统计就是按邮箱来统计的。
  2. 查看用户名和邮箱
    $ git config user.name
    $ git config user.email
    3.修改用户名和邮箱地址
    $ git config –global user.name “username”
    $ git config –global user.email “email”
  3. 配置文件在什么地方？
    位置1: /etc/gitconfig
    系统级别的配置,适用于所有的用户和所有的库,可以使用 $git config –system 来指定和修改,存储在Git安装目录下
    位置2: ~/.gitconfig
    用户级别的配置,适用于当前登录的用户,可以使用$git config –gloabal来指定和修改,存储在C:\Documents and Settings\$USER下
    位置3: .git/config
    库级别的配置,适用于某个具体的库,可以使用$git config来指定和修改,存储在具体的库隐藏的.git文件夹下
    说明:Git会优先使用库级别的配置,再然后是global级别的配置,最后是system级别的配置.

配置git ssh密钥

  1. 为什么要配置ssh密钥？
    由于本地Git仓库和GitHub仓库之间的传输是通过SSH加密的，所以必须要让github仓库认证你SSH key，在此之前，必须要生成SSH key。
    意思就是你要把文件传到我的服务器上，必须要经过一个配对码来建立连接，否则我也不认识你啊。

  2. 配置流程是怎样的？
    首先我们自己的电脑上得有一个公用的密钥，然后我们需要把电脑上的公钥告诉git。

2.1 查看自己电脑有没有公钥
默认情况下，用户的 SSH 密钥存储在其 ~/.ssh 目录下。 进入该目录并列出其中内容，你便可以快速确认自己是否已拥有密钥：
$ cd ~/.ssh
$ ls
authorized_keys2 id_dsa known_hosts
config id_dsa.pub
我们需要寻找一对以 id_dsa 或 id_rsa 命名的文件，其中一个带有 .pub 扩展名。 .pub 文件是你的公钥，另一个则是私钥。 如果找不到这样的文件（或者根本没有 .ssh 目录），你可以通过运行 ssh-keygen 程序来创建它们。
2.2 在本地配置公钥
首先 ssh-keygen 会确认密钥的存储位置（默认是 .ssh/id_rsa），然后它会要求你输入两次密钥口令。如果你不想在使用密钥时输入口令，将其留空即可。
$ ssh-keygen
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/liyan/.ssh/id_rsa):
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
Your identification has been saved in /Users/liyan/.ssh/id_rsa.
Your public key has been saved in /Users/liyan/.ssh/id_rsa.pub.
The key fingerprint is:
SHA256:uRx9/Xtr+MmHAIOLNT5BFWtkaNA9AUXkhran0iwKuBI liyan@Liyans-Mac.local
查看自己所生成的密钥：
$ cat ~/.ssh/id_rsa.pub
2.3 在git上填写公钥
进入 setting > SSH and GPG keys，点击New SSH key添加即可。

  1. 每次向git部署文件都需要密码怎么办？
    其实ssh-keygen的时候，可以输密码，也可以不输密码，那么如果你输了密码，ssh 登陆的时候还是会提示你将私钥的密码输入的(不需要输入服务器的密码)。而ssh-agent能避免这种反复输入私钥的烦恼。向ssh-agent添加私钥很简单：
    $ ssh-add ~/.ssh/id_rsa
    从ssh-agent中删除密钥：
    $ ssh-add -d ~/.ssh/id_xxx.pub

给博客配置主题

  1. 挑选主题
    在hexo官网上挑选主题，我选的是cactus主题。
  2. 将主题下载至本地
    点击主题名称，进入主题的git地址，下载并将该主题保存在网站本地文件的themes文件夹中，将文件夹名称命名为cactus。
  3. 配置_config.yml文件
    在配置文件 _config.yml 中将主题配置为theme: cactus。
    配置完成后生成文件时产生错误：
    hexo s
    INFO Start processing
    INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
    WARN No layout: index.html
    出现这个错误，检查以下原因：
  • 下载的主题是不是放在了themes文件夹下面，且主题文件夹的名称是否是minos。
  • 站点配置_config.yml中的主题名称是否正确。
    注意：有时因为主题本身的原因，可能在生成时产生诸如：
    Unhandled rejection ReferenceError: /Users/liyan/Holiyan/themes/minos/layout/index.ejs:4
    这样的错误，这可能是因为主题本身有问题，这时建议换个主题试试。

生成页面并在本地服务器查看

  1. 生成静态页面
    $ hexo g # hexo generate
  2. 在本地服务器查看
    $ hexo s # hexo server
    如果出现如下情况：
    hexo s
    FATAL Port 4000 has been used. Try other port instead.
    则表示端口被占用了，此时我们ctrl+c停止服务器，接着输入 $ hexo server -p 端口号来改变端口号。

域名解析

  1. 域名解析的目的
    我们申请了一个域名，当用户访问该域名时，域名指向哪里呢？解析的目的就是让域名指向页面。
  2. Github修改CNAME
    在github 你的pages库的 Setting 选项里，在Custom domain中输入你购买的域名，点击保存就可以了。
  3. 域名解析
    在阿里云域名管理中，点击解析，按照操作指引解析。
    做完以上三步就可以将域名指向github page了。
